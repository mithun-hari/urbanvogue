# UrbanVogue – Complete Project Analysis & Workflow

## 1. Architecture Overview
```
Client ──► API Gateway (8080) ──► 6 Micro‑services
```
- **API Gateway** (Spring Cloud Gateway) routes requests based on URL prefixes.
- Each micro‑service runs on its own port and owns its own MySQL database.

| Service | Port | Base Path (via gateway) | Database |
|---|---|---|---|
| Auth | 8082 | `/api/auth/**` | `urbanvogue_auth` |
| Product | 8083 | `/api/products/**` | `productdb` |
| Order | 8085 | `/orders/**` | `urbanvogue_order` |
| Inventory | 8086 | `/api/inventory/**` | `urbanvogue_inventory` |
| Payment | 8087 | `/payments/**` | `urbanvogue_payment` |
| Notification | 8088 | `/api/notifications/**` | — |

## 2. Service Responsibilities
| Service | Core Domain | Key Endpoints | Security |
|---|---|---|---|
| **Auth** | User registration & login | `POST /api/auth/register`, `POST /api/auth/login` | JWT issued on login; filter validates token on every request. |
| **Product** | CRUD for products | `GET /api/products`, `POST /api/products` (ADMIN) | JWT required; `@PreAuthorize("hasRole('ADMIN')")` protects writes. |
| **Inventory** | Stock levels per product | `GET /api/inventory/{productId}`, `POST /deduct`, `POST /restore` | Currently **open** (`permitAll`). |
| **Order** | Orchestrates order creation, status, retrieval | `POST /orders`, `GET /orders/{id}`, `PUT /orders/{id}/status` | Open (no auth). |
| **Payment** | Stripe checkout, webhook handling, order‑status update, email trigger | `POST /payments`, `POST /payments/webhook` | Open (no auth). |
| **Notification** | Sends email via Gmail SMTP | `POST /api/notifications/send-email` | No security configured. |

All services use Spring Data JPA, Lombok, and MySQL.

## 3. Inter‑service Communication
| From → To | Technology | Purpose |
|---|---|---|
| Order → Product | OpenFeign | Fetch product details & price. |
| Order → Inventory | OpenFeign | Verify stock, deduct, restore on rollback. |
| Order → Payment | WebClient (reactive) | Initiate Stripe checkout session. |
| Payment → Order | OpenFeign | Update order status to `PAID` after webhook. |
| Payment → Notification | OpenFeign | Send receipt email after successful payment. |

## 4. End‑to‑End Order Flow
1. **Register / login** → JWT returned.
2. **Browse products** (`GET /api/products`).
3. **Create order** (`POST /orders`) with product IDs & quantities.
4. **Order Service**:
   - Calls **Product Service** for pricing.
   - Calls **Inventory Service** to check & deduct stock.
   - Persists order with status `CREATED`.
   - Calls **Payment Service** → Stripe Checkout Session created → URL returned.
5. **Customer** completes payment on Stripe.
6. Stripe posts **webhook** to **Payment Service** (`/payments/webhook`).
7. **Payment Service** validates webhook, marks payment `SUCCESS`, then:
   - Calls **Order Service** → set order status `PAID`.
   - Calls **Notification Service** → send confirmation email.
8. If payment fails, **Payment Service** tells **Order Service** to set status `FAILED` and the **Order Service** restores inventory.

## 5. Security Snapshot
- **JWT** (HS256, shared secret) generated by Auth Service, validated by Product Service (hard‑coded same secret).
- Auth Service protects all non‑public endpoints; Product Service protects write ops via `@PreAuthorize`.
- Inventory, Order, Payment, Notification currently allow all requests – they should be hardened (e.g., reuse JWT filter).

## 6. Database Schemas
### `urbanvogue_auth` – Users
| Column | Type | Constraints |
|---|---|---|
| `id` | BIGINT | PK, auto‑increment |
| `username` | VARCHAR | Unique, not null |
| `email` | VARCHAR | Unique, not null |
| `password` | VARCHAR | BCrypt hash |
| `role` | VARCHAR | `USER` / `ADMIN` |
### `productdb` – Products
| Column | Type |
|---|---|
| `id` | BIGINT PK |
| `name` | VARCHAR |
| `description` | VARCHAR |
| `price` | DOUBLE |
### `urbanvogue_order` – Orders & Items
- **orders**: `id`, `user_id`, `total_amount`, `status`, `created_at`
- **order_items**: `id`, `product_id`, `quantity`, `price`, `order_id`
### `urbanvogue_inventory` – Inventory
| Column | Type |
|---|---|
| `id` | BIGINT PK |
| `product_id` | BIGINT |
| `available_quantity` | INT |
| `reserved_quantity` | INT |
### `urbanvogue_payment` – Payments
| Column | Type |
|---|---|
| `id` | BIGINT PK |
| `order_id` | BIGINT |
| `amount` | DOUBLE |
| `payment_method` | VARCHAR |
| `payment_status` | VARCHAR (`PENDING`/`SUCCESS`) |
| `transaction_id` | VARCHAR |
| `stripe_session_id` | VARCHAR |
| `created_at` | DATETIME |

## 7. Technology Stack Summary
- **Java 17**, **Spring Boot 3.5.x**
- **Spring Cloud Gateway** (WebMVC)
- **Spring Security** + **JWT (jjwt)**
- **OpenFeign** & **WebClient** for inter‑service calls
- **Stripe Java SDK** for payments
- **Spring Mail** (Gmail SMTP) for emails
- **MySQL**, **Spring Data JPA**, **Hibernate**
- **Maven**, **Lombok**, **SLF4J/Logback**
